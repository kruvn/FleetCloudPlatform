# Contributing

Thank you for your interest in Fleet Cloud Platform.

Although this project is currently maintained by a single developer, all development follows the same engineering standards used for collaborative projects.

## Development Principles

- Architecture First
- API First
- Event Driven
- Documentation before implementation
- Small incremental changes
- Every feature must have a clear purpose

---

## Branch Strategy

main

Production-ready code.

feature/*

New functionality.

bugfix/*

Bug fixes.

research/*

Experiments and spikes.

---

## Commit Messages

The project follows Conventional Commits.

Examples

feat: add telemetry gateway

fix: resolve duplicate event processing

docs: update vision

refactor: simplify mission aggregate

---

## Pull Requests

Even when working alone, features should be developed as if they were submitted for review.

Every completed feature should include

- updated documentation
- tests (where applicable)
- changelog entry (when needed)
