# ADR-0001: Event Driven Architecture

## Status

Accepted

---

# Context

Fleet Cloud Platform must process large amounts of data generated by autonomous devices.

The system contains multiple independent domains:

- Fleet
- Missions
- Telemetry
- Notifications
- Analytics
- Billing

Direct synchronous communication between all services would create strong coupling and reduce scalability.

---

# Decision

The platform will use Event-Driven Architecture as the primary communication model.

Services will publish domain events and consume events from the event platform.
